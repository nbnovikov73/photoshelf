# Product Principles

1. Speed beats feature count.
2. Ownership beats reach.
3. Archive beats feed.
4. Selection beats volume.
5. Calm beats engagement.
6. Mobile publishing is the core workflow.
7. Photographs are primary; interface is secondary.
8. Public site should look like an editorial object, not an admin export.
9. Every new feature must defend its existence.
10. Drafts are private by default.
