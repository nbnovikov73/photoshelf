# UX Bible

## Primary UX rule

The upload-to-publish flow must feel shorter than opening a CMS.

Target: `Open admin → Upload → Preview → Publish`.

Admin should be mobile-first, thumb-friendly, visually quiet, direct, and forgiving.

Avoid multi-step wizards, dense dashboards, configuration-heavy forms, unnecessary confirmations, and desktop-only workflows.

Public pages should feel like a photo book, gallery wall, or editorial portfolio.
